Zero-touch daily compliance:
How I automated EOD tracking for 70+ VAs

A 3-phase N8N + Claude AI system that runs every weekday without my intervention — from morning reminders to AI classification to overnight catch-up and a daily executive summary in my inbox.

RoleSole architect & operator
StackN8N Cloud, Gmail API, Sheets, Claude
Workflows3 N8N flows, 1 dashboard
StatusDaily in production since Mar 2026

TL;DR

Tracking 70+ daily End-Of-Day reports manually meant being online at cutoff every single day. I built a 3-phase N8N Cloud system: morning nudge, evening Gmail scan with AI classification, and overnight catch-up with auto-drafted escalations. Now I read a daily summary email at 9am instead of running compliance for an hour every night.

The problem

Every workday, 70+ VAs send an end-of-day report. The manager (me) needs to know: who submitted, who didn't, and who was late. The old workflow:

  1. Be online at the 7pm cutoff
  2. Open Gmail, search 3–4 different terms because EOD subject lines are inconsistent
  3. Cross-reference against the active VA roster (which changes weekly)
  4. Manually mark on-time / late / missing in a Google Sheet
  5. Draft escalation emails to no-shows
  6. Send a daily summary to my line manager

Total: ~45–60 minutes a night, 5 nights a week. Plus the cognitive overhead of "did I do compliance yet?" all evening. Plus the fact that one missed night meant playing catch-up the next morning.

The architecture

I split the day into three phases, each as its own N8N Cloud workflow on a cron schedule. Phases share state through a single Google Sheet that doubles as the live dashboard.

PHASE 1

Morning Reminder

9:30 AM weekdays

Pulls today's active VA roster from the Google Sheet, builds a personalized reminder email, sends to all VAs in one batch. Sets the daily compliance row to status=“awaiting”.

PHASE 2

Cutoff Scan + AI Classification

7:00 PM weekdays

Runs three parallel Gmail searches with different query patterns (EOD subjects vary), merges and deduplicates the results, then sends each match to Claude with the VA name & expected schedule. Claude classifies as On-Time / Late / Missing with a 1-line reason. Results write back to the dashboard.

PHASE 3

Overnight Catch-Up + Daily Report

9:30 AM next day

Re-scans Gmail for late submissions that arrived overnight, reclassifies them, updates the dashboard. Drafts escalation emails for VAs still missing (sits in my drafts folder for one-click send). Builds an HTML executive summary with on-time/late/missing rates and emails it to me.

The classifier

The hard part isn't reading email — it's deciding which emails count. EOD subject lines are inconsistent ("EOD Report", "End of Day", "EOD - Apr 17", "Daily Update", "Today's report"). Senders use different addresses. Some forward through clients. Time zones differ.

Rather than maintaining a regex zoo, I let Claude do the classification with structured input:

Input:
{
  "va_name": "Maria S.",
  "expected_send_window": "5:00pm-7:00pm AEST",
  "received_at": "2026-04-17T19:14:00+10:00",
  "subject": "EOD - April 17",
  "body_preview": "Hi team, here's my report for today..."
}

Claude returns:
{
  "status": "Late",
  "minutes_past_cutoff": 14,
  "confidence": "high",
  "reason": "Submitted 14 minutes after cutoff window"
}

This lets the model handle the messy edge cases (forwarded mails, weird subjects, time-zone math) without me hand-coding every rule. The N8N flow batches submissions into chunks of ~10 to keep token costs predictable.

Why N8N over a Python script? Three reasons. (1) No PC dependency — runs in N8N Cloud, so my laptop being off doesn't matter. (2) Native Gmail/Sheets nodes — auth and API quirks handled. (3) Visual debugging — when something breaks, I can see exactly which node failed and what it received.

The dashboard

One Google Sheet, three tabs: Roster (active VAs + schedules), Daily Log (every classification with timestamp), Dashboard (rolling on-time / late / missing rates with sparklines). Stakeholders open one URL; nothing to install.

What I'd build differently

Roster sync, not roster file

I keep the active VA roster in a Sheet tab. Every onboard/offboard means manually updating it. Next version: pull live from our HRIS so the system always knows who's active.

Slack DM channel for nudges

Email reminders work but get lost. Posting a personalized "you haven't sent your EOD yet" DM in Slack would land harder. Easy add via N8N Slack node.

Anomaly detection on the side

The system catches missing reports. It doesn't catch thin reports (a one-line EOD passes "On-Time"). A second classifier pass on report quality would catch the "compliance theater" pattern.

Rolling weekly digest for VAs

Currently only I see the dashboard. A weekly per-VA "your on-time rate is 94% — nice work" email would close the feedback loop without me chasing.

Stack used

  • N8N Cloud — orchestration, all 3 phases as separate workflows on cron
  • Gmail API — via N8N's native node (3 parallel queries per scan)
  • Google Sheets API — roster, daily log, and live dashboard
  • Claude Sonnet 4.6 via OpenRouter — classification with structured outputs
  • HTML email templates — daily executive summary built inline in N8N
